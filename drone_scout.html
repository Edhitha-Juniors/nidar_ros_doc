Module contents

The drone_scout package provides a complete ROS 2 software stack for autonomous drone surveillance, inspection, and data collection. Developed specifically for NIDAR Competition.

It is designed as an end-to-end system that supports the full operational pipeline — from mission planning and autonomous navigation to real-time perception, geotagging, target management, and logging.

This package enables drones to autonomously survey an area, detect and localize objects of interest, and maintain mission-level awareness through centralized monitoring and coordination.

Key Modules & Nodes

  • waypoint_generate Generates autonomous survey missions, geofences, and flight boundaries from KML inputs.

  • mission_commander Executes missions by interfacing with MAVROS for flight control, waypoint handling, and mission automation.

  • Camera Nodes (cam_csi, cam_usb, etc.) Support image capture across multiple camera hardware platforms.

  • geotag Associates captured imagery with GPS and IMU telemetry for accurate spatial tagging.

  • inference Runs real-time YOLO-based object detection on incoming camera frames.

  • drone_status A centralized health and diagnostics node for monitoring system components, mission state, and runtime performance.

  • cluster_delivery Delivery-drone-specific control module (Barbarik), enabling target execution and payload delivery based on central drone commands.

  • cluster Central-drone coordination node that performs live target clustering, prioritization, and multi-drone target assignment.

  • mapping Experimental SUAS mapping integration using Docker-based OpenDroneMap (ODM) workflows.

  • mavlink_commands Provides utilities for advanced MAVLink operations such as fence uploads, system tuning, and custom command execution.

  • precision_repo In-house visual precision correction framework for achieving centimeter-level localization accuracy.

  • precision_server Backend service responsible for controlling and deploying precision correction capabilities.

  • target_manager Maintains mission target state via a Flask-based server, ensuring synchronization and coordination between multiple drones.

Typical Workflow

A standard mission flow includes:

  1. Mission Planning using waypoint_generate

  2. Autonomous Execution through mission_commander

  3. Real-Time Data Capture via camera nodes

  4. Geotagging + Detection with geotag and inference

  5. Target Tracking and Coordination through clustering and management services

Together, these nodes form a modular and scalable architecture for intelligent aerial scouting and autonomous drone operations.
