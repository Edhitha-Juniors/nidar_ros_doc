drone_scout.inference module

drone_scout.yolo_inference

This module contains the YOLO Inference Node.

It performs:

  • Real-time object detection from onboard camera feeds

  • Filtering for relevant target classes (e.g., person)

  • Publishing detection results to ROS topics

  • Logging detections for clustering

Published Topics

  • /yolo_results_topic

This node provides the perception layer of the drone swarm.

class drone_scout.inference.YoloInferenceNode

Bases: Node

Performs YOLO inference on images and calculates GPS coordinates for detections.

This node subscribes to a topic providing image metadata, including the drone’s GPS position and yaw. For each message, it loads the corresponding image, runs YOLO detection, filters detections based on class and size, applies Non-Max Suppression, and then calculates the estimated latitude and longitude of each final detection. Results, including cropped images and a master CSV file, are saved to the filesystem, and individual detection data is published.

Parameters:

  • folder_path (str) – The base directory containing the ‘captured_frames’ folder and for saving outputs.

  • yolo_model_path (str) – The file path to the YOLOv8 model weights (.pt file).

  • conf_threshold (float) – The minimum confidence score to consider a detection.

  • box_threshold (float) – A threshold for filtering boxes (not used in current logic but declared).

  • max_area_ratio (float) – The maximum ratio of a bounding box’s area to the total image area to be considered valid.

  • IOU_THRESHOLD (float) – The Intersection over Union (IoU) threshold for Non-Max Suppression (NMS).

  • gsd (float) – Ground Sample Distance in cm/pixel. Crucial for calculating real-world coordinates.

Subscriptions:
/geotag_data (std_msgs.msg.String): Receives comma-separated metadata:

“timestamp,lat,lon,alt,orientation_w,yaw,filename”.

Publications:

/yolo_results_topic (std_msgs.msg.String): Publishes a JSON string for each valid detection.

geotag_callback(msg)

Callback function for the geotag message subscriber. Parses the geotag message, loads the image, performs YOLO inference, and publishes the results.

get_coordinates(image_path, point2, point1, lat, lon, yaw)

Calculates the GPS coordinates of a detected object.

Parameters:

  • image_path – Path to the full image (for logging).

  • img_center_xy – A tuple (x, y) of the image’s center pixel.

  • crop_center_xy – A tuple (x, y) of the detected object’s center pixel.

  • lat – The latitude of the image center (drone’s position).

  • lon – The longitude of the image center (drone’s position).

  • yaw – The yaw of the drone in degrees.

Returns:

A tuple containing the calculated (latitude, longitude) for the object, or (None, None) on failure.

newlatlon(lat, lon, hdg, dist, movementHead)

Projects a new lat/lon point from a start point, bearing, and distance.

Parameters:

  • lat – Starting latitude in decimal degrees.

  • lon – Starting longitude in decimal degrees.

  • drone_hdg – The drone’s heading (yaw) in degrees.

  • dist_m – The distance to the new point in meters.

  • movement_hdg – The bearing from the start point to the new point, relative to the image frame, in degrees.

Returns:

A tuple of the new (latitude, longitude) in decimal degrees.
